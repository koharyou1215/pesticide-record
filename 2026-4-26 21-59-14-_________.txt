アプリ概要

農薬の希釈計算と使用量計算を多角的に行うアプリケーションです。農薬の登録機能を持ち、初期データとして主要な殺菌剤・殺虫剤48種のデータを保持しています。また、よく使う任意の農薬の情報を保存・呼び出しできます。

主要機能一覧

1. 計算機能

タブ切り替えで以下の3つの計算モードを提供します。

原液量計算 (希釈計算)
入力: 水の量 (L/mL)、希釈倍数
オプション入力: 登録農薬の選択 (選択時は希釈倍数を自動入力)
出力: 必要な原液量 (mL)、希釈液の総量 (L/mL)
面積計算 (面積あたりの使用原液量計算)
入力: 10aあたり使用量 (mL/g)、対象面積 (アール)
オプション入力: 登録農薬の選択 (選択時は使用量と単位を自動入力)
出力: 使用原液量 (mL/g)
水量計算 (原液量から水量を計算)
入力: 原液の量 (mL/g/L/kg)、希釈倍数
オプション入力: 登録農薬の選択 (選択時は希釈倍数を自動入力)
出力: 必要な水の量 (L/mL)、希釈液の総量 (L/mL)
2. 農薬管理機能

初期データ: 主要な農薬情報 (48種類、殺菌剤・殺虫剤) を初期設定として内包（アプリ初回起動時にローカルストレージに保存）。
登録済み農薬の検索・一覧:
病害虫名でのドロップダウン絞り込み検索。
農薬の一覧表示 (名前、グループ、対象病害虫、使用量・希釈倍数)。
計算への反映: 一覧画面の「反映」ボタンを押すことで、現在開いている計算タブ（原液量・面積・水量）にその農薬を選択状態として即座に適用し、ページ上部へ自動スクロール。
削除: 登録済み農薬の個別削除。万が一全て削除した場合でも、リロード（あるいはキャッシュクリア）で初期データが復活する仕組みを備えています。
新規登録:
入力項目: 農薬名 (必須)、グループ、効果のある病害虫 (複数選択可)、10aあたり使用量、単位 (mL/g)、標準希釈倍率。
保存先: ブラウザのローカルストレージ (localStorage: registeredPesticides)。
3. その他機能

入力値リセット: 画面下部の「入力をリセット」ボタンで、全ての計算フォームと検索条件を初期状態に戻す。
単位の動的フォーマット: 計算結果は、数値の大きさに応じて L / mL を自動的・直感的にフォーマットして表示します（例: 1 L 500 mL）。
技術仕様

フロントエンド: React 19, TypeScript, Vite
スタイリング: ピュア CSS (レスポンシブデザイン対応、CSSアニメーション活用)
データ永続化: Web Storage API (localStorage

こういうアプリをつくろうとおもっています。

## 1. Webアプリとして公開する

一番簡単です。

- iPhone: Safari / Chrome で開ける
- Android: Chrome で開ける

- URLを送れば誰でも使える

今の React + Vite 構成なら、この方法が最短です。

## 2. PWA化する

おすすめです。

Webアプリですが、スマホのホーム画面に追加できます。

- iPhone: Safari の「ホーム画面に追加」
- Android: Chrome の「ホーム画面に追加」
- アプリっぽく全画面起動できる
- App Store / Google Play 審査なし
- localStorage も使える
- オフライン対応も可能

この農薬計算機なら、まず **PWA対応Webアプリ**が一番現実的です。

## 3. ネイティブアプリ化する

本物のスマホアプリとして配布する方法です。

候補:

- Capacitor
- React Native
- Flutter

ただし、

- iPhoneは Apple Developer Program が必要
- App Store 審査がある
- Androidも Google Play Console 登録が必要
- 配布・更新の手間が増える

個人・小規模利用なら最初からこれは重いです。

>> USER: これが試作なんだけど、これってわざわざ3個のタブに分けなければいけないかな？いずれも変数で一つの表で解決できない？

>> ASSISTANT: 判定  
**3タブ必須ではありません。**  
むしろこのアプリなら、最終的には **1つの計算フォームに統合した方が使いやすい可能性が高い**です。

理由は、3つの計算は全部これで表せるためです。

```text
水量 × 1000 ÷ 希釈倍数 = 原液量
原液量 × 希釈倍数 = 水量mL
10a使用量 × 面積a ÷ 10 = 面積分の使用量
```

## 結論

今の3タブ構成より、次の形がよいです。

```text
1画面 + 計算タイプ選択 + 必要項目だけ表示
```

完全な表形式より、スマホではこの方がミスが少ないです。

仕様変更するならこうです。

```text
計算UIは3タブ方式ではなく、単一の計算カードに統合する。
上部に「計算したいもの」を選ぶセグメントボタンを配置する。

計算タイプ:
- 原液量を求める
- 水量を求める
- 面積分の使用量を求める

選択された計算タイプに応じて、必要な入力欄のみ表示する。
農薬を選択した場合は、希釈倍数または10a使用量を自動入力する。
結果は同じカード内の結果エリアに大きく表示する。

内部計算ロジックは共通化し、各変数から必要な値を算出する。
```
